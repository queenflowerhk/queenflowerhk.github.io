A flat, zero-build static site deployed to `queenflowerhk.github.io` on GitHub Pages. The entire application lives in one file — `docs/index.html` — which bundles HTML sections (ceremonial, funeral, wreaths, opening, association, graduation, pets, delivery, about, contact), Tailwind CSS via CDN with an inline `tailwind.config` theme override, Google Fonts (Noto Serif/Sans TC + Playfair Display + Inter), and Font Awesome icons. Client-side state is implemented in vanilla JavaScript inside the same file: a `translations` dictionary drives zh/en i18n through `data-i18n` attributes, a shopping-cart sidebar persists items in memory and renders a WhatsApp checkout link, and product grids are populated from JS data arrays into DOM containers (`#ceremonial-grid`, `#funeral-products-grid`, etc.). A `docs/.nojekyll` marker tells GitHub Pages to skip Jekyll processing so the raw HTML is served as-is. There is no build pipeline, framework, or server — it is pure client-side HTML/CSS/JS.
